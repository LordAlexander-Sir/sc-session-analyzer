Setup Guide

Getting started takes about two minutes. Follow these steps in order.

Step 1
Install the Extension
  1. Download the SCHangarExporter folder from your server admin or the Discord server download channel.
  2. Open Chrome and navigate to chrome://extensions
  3. Enable Developer mode using the toggle in the top-right corner.
  4. Click Load unpacked and select the SCHangarExporter folder you downloaded.
  5. After it loads, click the puzzle piece 🧩 icon in Chrome's toolbar, find SC Hangar Exporter, and click the pin 📌 to keep the 🚀 icon visible in your toolbar.
💡 The extension works in Chrome-based browsers (Chrome, Edge, Brave). It will not work in Firefox or Safari.
Step 2
Connect Your Discord Account
  1. Click the 🚀 icon in your Chrome toolbar to open the extension.
  2. Click Connect Discord Account.
  3. A Discord authorization tab will open — click Authorize.
  4. The tab closes automatically and your Discord username will appear in the extension.
💡 The extension only reads your Discord username and ID to identify you to the bot. It cannot see your messages, servers, or any other account information. That's the only setup you need — the extension talks to the bot automatically, with no keys or webhooks to configure.
Step 3
Point It to Your Log Files (optional but recommended)

This lets the extension include your session history — missions, blueprints, combat, and more — alongside your hangar data when you send to the bot.

  1. Click 📂 Set Log Folder in the extension.
  2. Navigate to and select your Star Citizen log backups folder:
C:\Program Files\Roberts Space Industries\StarCitizen\LIVE\logbackups\

If Star Citizen is installed on a different drive, open the RSI Launcher → click the gear ⚙️ next to LIVE → Show game folder, then open the logbackups\ subfolder from there.

Step 4
You're Ready — Fetch and Send
  1. Make sure you are logged into robertsspaceindustries.com in any Chrome tab.
  2. Click the 🚀 extension icon and hit 🔄 Fetch Hangar Data. The extension will pull your ships, paints, and pledges directly from RSI.
  3. Once complete, you have two options:
    • 📊 Open in Analyzer — opens your data in this analyzer instantly, no file needed.
    • 📤 Send to Bot — sends everything straight to the bot, which updates your fleet profile and replies with a summary right in the extension.
💡 You can use both options. Open in Analyzer to explore your data, then Send to Bot to update your profile — or do them in either order.

Error Code Reference

When the extension encounters a problem it shows a clickable error code. Find your code below for the fix.

EXT-001 Not logged into RSI
The extension tried to fetch your hangar but RSI returned a login page instead of your ships.
How to fix
  1. Open robertsspaceindustries.com in any Chrome tab and sign in.
  2. Return to the extension and click 🔄 Fetch Hangar Data again.
EXT-002 Could not reach RSI page
The extension opened an RSI tab but the page didn't respond in time, or the tab was already on a page the extension can't reach.
How to fix
  1. If you have an RSI tab open, refresh it and wait for it to fully load.
  2. Click 🔄 Fetch Hangar Data again.
  3. If this repeats, make sure you are logged into RSI — see EXT-001.
EXT-003 RSI page timed out
The RSI account page took longer than 30 seconds to respond. Usually a slow connection or RSI server issue.
How to fix
  1. Check your internet connection.
  2. Try opening robertsspaceindustries.com/account manually — if it loads slowly, RSI may be having issues.
  3. Wait a minute and try again.
EXT-005 No log files found
The folder you selected doesn't contain any Star Citizen log files, or the wrong folder was selected.
How to fix
  1. Click 📂 Set Log Folder again.
  2. Navigate to and select:
    C:\Program Files\Roberts Space Industries\StarCitizen\LIVE\logbackups\
  3. If SC is on a different drive, open the RSI Launcher → gear ⚙️ → Show game folder → open the logbackups\ subfolder.
  4. Make sure you select the logbackups folder itself, not the LIVE folder above it.
EXT-009 Analyzer auto-load failed
The extension couldn't push data directly into the analyzer tab. A backup JSON file was downloaded automatically so you don't lose anything.
How to fix
  1. Find the downloaded sc-hangar-export-*.json file in your Downloads folder.
  2. Open the analyzer in a new tab.
  3. Drag the JSON file onto the analyzer's drop zone — it loads the same way.
If this happens every time, a browser extension or Brave Shield may be blocking scripts on the analyzer page — see Browser Blocking.
Send to Bot didn't work
If sending fails, the extension shows a short message explaining why. The common ones:
  • "Could not reach the bot" — the bot is offline or restarting. Wait a moment and try again.
  • "Your Discord session expired" — reconnect your Discord account (it disconnects automatically), then send again.
  • "Export is too large" — usually a very large log history. Try clicking Send again without setting a log folder, or with fewer logs.

Browser Blocking Issues

Some browsers and ad blockers block the extension from automatically loading your data into the analyzer. A backup JSON file is always downloaded when this happens so you never lose your data — just drag it onto the analyzer's drop zone.

EXT-011 Brave Shield blocking the analyzer
Brave's built-in shield is blocking script injection on the analyzer page.
How to fix
  1. Click the 🦁 lion icon in Brave's address bar while the analyzer tab is open.
  2. Set Shields to Down for this site, or click Advanced controls → set Scripts to Allow all scripts.
  3. Reload the analyzer tab, then click 📊 Open in Analyzer in the extension again.
💡 This only affects the analyzer site — lowering the shield here doesn't change your protection anywhere else.
EXT-012 Ad blocker blocking the analyzer
An ad blocker or privacy extension (uBlock Origin, Privacy Badger, etc.) is blocking script execution on the analyzer page.
How to fix
  1. Click your ad blocker's icon in the toolbar while the analyzer tab is open.
  2. Add the analyzer site to its allowlist, or temporarily disable it for that tab.
  3. Reload the analyzer tab and try again.
💡 Alternatively, use the downloaded JSON backup — drag it onto the analyzer's drop zone and everything loads just the same.

Admin Setup

Good news: there's almost nothing to do. Imports go straight to the bot, so there are no webhooks, import channels, or keys to set up anymore.

What members need

  1. The bot must be in your Discord server, and the member must be a member of that server.
  2. The member installs the extension and clicks Connect Discord Account — that's the whole setup.
💡 The bot identifies each member from their Discord login and matches them to your server automatically. There are no import keys or webhook URLs to distribute or manage.

Optional: the in-Discord Fleet Dashboard

If you want the Fleet Dashboard inside Discord — for viewing profiles, leaderboards, and the org fleet — run /setupdashboard in the channel where you'd like it. This is optional and is not required for hangar imports to work.

FAQ

Does the extension send my data anywhere automatically?

No. Nothing happens unless you click a button. Data only leaves your device when you click Send to Bot (sends it to the bot) or Download JSON (saves to your Downloads folder). Open in Analyzer pushes data directly to the browser tab — nothing goes to any server.

What exactly does the bot receive?

When you click Send to Bot, the extension sends: your ships, paints, blueprints, session stats (missions, kills, deaths, aUEC earned), and hardware info (CPU, GPU, RAM) from your log files. It's sent over an encrypted (HTTPS) connection straight to the bot — it never passes through a Discord channel, so no one else can see or download it.

Who can see my data?

Other members of your server can always see your ships, paints, and blueprints — that's the point of the app. Your game stats are shared by default, while your hardware specs and detailed session history are hidden by default. You control all of this in Discord: open Fleet Dashboard → Personal Profile → 🔒 Privacy and toggle each one. You always see your own full profile — these settings only change what other members see.

How do I delete my data?

In Discord, open Fleet Dashboard → Personal Profile → 🗑️ Delete My Data. It permanently removes everything the bot stores about you in that server — ships, paints, pledges, blueprints, stats, hardware info, and session history. This can't be undone, but you can re-import any time from the extension.

Can someone spoof an import as me?

No. Every import is authenticated with your Discord login — the bot confirms your identity with Discord on each upload, so only you can update your own profile. There's no key to leak or manage. If you're ever concerned, just click Disconnect in the extension; reconnecting starts a fresh session.

My Send to Bot button is greyed out

Two things are needed before you can send: your hangar data must be fetched, and your Discord account must be connected. Hover over the button to see which one is still missing.

Where do my import results show up?

Right in the extension. After you click Send to Bot, it shows a summary of exactly what was imported — ships added, sessions stored, and so on. You don't need to check Discord or wait for a DM.

The extension worked before but now shows errors

RSI occasionally updates their website, which can affect how the extension reads your hangar. Check for an extension update in the Chrome Web Store. If Discord Connect fails, try disconnecting and reconnecting — your session may have expired.

I don't see a Fleet Dashboard in Discord

The Fleet Dashboard is only available in servers that have the bot installed and configured. If you don't see it, ask your server admin whether the bot has been set up.

Setup Guide

① Install the Extension

  1. Download the SCHangarExporter folder from your server admin or the project repo.
  2. Open Chrome and navigate to chrome://extensions
  3. Enable Developer mode (toggle, top right).
  4. Click Load unpacked and select the SCHangarExporter folder.
  5. The 🚀 rocket icon appears in your Chrome toolbar. Pin it for easy access.

② Log into RSI

Sign into robertsspaceindustries.com in any Chrome tab. The extension finds your session automatically — you don't need to be on that tab when you click Fetch.

③ Connect Discord

  1. Open the extension popup and click 🔗 Connect Discord Account.
  2. A Discord authorization tab opens. Click Authorize.
  3. The tab closes automatically and you'll see your Discord username in the popup.
💡 The extension only requests identify scope — it reads your Discord username and ID, nothing else. No messages, no server management.

④ Set your Bot Webhook URL

Your server admin creates a webhook for the import channel. Open 🤖 Bot Webhook URL in the extension and paste it in. You only do this once.

Admins: Discord → your channel → Edit Channel → Integrations → Webhooks → New Webhook → Copy URL.

⑤ Set your Import Key

  1. In Discord, open the Fleet Dashboard and click Personal Profile.
  2. Click 📤 Import to see your personal import key.
  3. Copy the key and paste it into the extension's 🔑 Bot Import Key settings.
💡 Keep this key private. If you think it was compromised, click 🔄 Generate New Key in Discord and paste the new one into the extension.

⑥ Set your Log Folder (optional)

Click 📂 Set Log Folder and navigate to:

C:\Program Files\Roberts Space Industries\StarCitizen\LIVE\logbackups\

If Star Citizen is on a different drive, open the RSI Launcher → gear ⚙️ next to LIVE → Show game folder, then open the logbackups\ subfolder.

Error Code Reference

When the extension encounters an error it displays a clickable code like [EXT-001]. Click it or find the code below for the fix.

EXT-001 Not logged into RSI
The extension fetched your hangar but RSI returned a login page instead of your ships.
How to fix
  1. Open robertsspaceindustries.com in any Chrome tab.
  2. Sign in with your RSI account.
  3. Return to the extension and click 🔄 Fetch Hangar Data again.
EXT-002 Could not reach RSI content script
The extension opened an RSI tab but the page didn't load in time for the content script to start, or the tab was already on a page the extension can't reach.
How to fix
  1. If you have an RSI tab open, refresh it and wait for it to fully load.
  2. Click 🔄 Fetch Hangar Data again — the extension will try a fresh background tab.
  3. If this repeats, check that you're logged into RSI (EXT-001).
EXT-003 RSI page timed out
The RSI account page took longer than 30 seconds to load. Usually a slow connection or RSI server issue.
How to fix
  1. Check your internet connection.
  2. Try opening robertsspaceindustries.com/account manually — if it loads slowly, RSI may be having issues.
  3. Wait a minute and try again.
EXT-005 No .log files found
The folder you selected doesn't contain any Star Citizen log files, or you selected the wrong folder.
How to fix
  1. Click 📂 Set Log Folder again.
  2. Navigate to:
    C:\Program Files\Roberts Space Industries\StarCitizen\LIVE\logbackups\
  3. If SC is on a different drive, open the RSI Launcher → gear ⚙️ → Show game folder → open logbackups\.
  4. Make sure you select the logbackups folder itself, not a subfolder or the LIVE folder.
EXT-007 No webhook URL set
Send to Bot requires a Discord webhook URL to know which channel to post to.
How to fix
  1. Open the 🤖 Bot Webhook URL section in the extension.
  2. Paste the webhook URL your server admin provided.
  3. Click Save Webhook URL.
Admins: Discord → channel → Edit Channel → Integrations → Webhooks → New Webhook → Copy URL.
EXT-008 Import Key not set
The bot requires a personal import key to verify that uploads came from you. The extension doesn't have one saved yet.
How to fix
  1. Open your Discord server and go to the Fleet Dashboard.
  2. Click Personal Profile, then 📤 Import.
  3. Copy the import key shown there.
  4. In the extension, open 🔑 Bot Import Key and paste it in.
  5. Click Save Import Key.
EXT-009 Analyzer auto-load failed
The extension couldn't inject data into the analyzer tab. A backup JSON file was downloaded automatically.
How to fix
  1. Find the downloaded sc-hangar-export-*.json file in your Downloads folder.
  2. Open the analyzer in a new tab.
  3. Drag the JSON file onto the analyzer's drop zone.
If this happens every time, a browser extension may be blocking scripts — see EXT-012.
EXT-010 Webhook post failed
The extension posted to the Discord webhook but Discord returned an error. The error code and message are shown alongside this code.
Common causes
  • 400 — Malformed request. Try again; if it persists, contact your admin.
  • 401 / 403 — Webhook URL may be invalid or deleted. Ask your admin for a new one.
  • 404 — Webhook no longer exists. Ask your admin to create a new one.
  • 429 — Rate limited. Wait a minute and try again.

Browser Blocking Issues

Some browsers and extensions block scripts on external sites, which prevents the extension from automatically loading your data into the analyzer. A backup JSON file is always downloaded when this happens so you never lose your data.

EXT-011 Brave Shield blocking the analyzer
Brave's built-in shield is blocking script injection on the analyzer page.
How to fix
  1. Click the 🦁 lion icon in Brave's address bar while the analyzer tab is active.
  2. Change Shields to Down for this site, or click Advanced controls → set Scripts to Allow all scripts.
  3. Reload the analyzer tab.
  4. Click 📊 Open in Analyzer in the extension again.
💡 This only affects the analyzer site — lowering the shield here doesn't change your protection on other sites.
EXT-012 Browser extension blocking scripts
An ad blocker or privacy extension (uBlock Origin, Privacy Badger, etc.) is blocking script execution on the analyzer page.
How to fix
  1. Click your ad blocker's icon in the toolbar while the analyzer tab is active.
  2. Add the analyzer site to its allowlist / whitelist, or temporarily disable it for that tab.
  3. Reload the analyzer tab and try again.
💡 Alternatively, use the downloaded JSON backup — drag it onto the analyzer's drop zone and it loads the same way.

Bot Setup (Server Admins)

Create the import channel

  1. Create a Discord channel for imports (e.g. #fleet-imports). It can be private — only the bot needs access.
  2. Give the bot the following permissions in that channel: View Channel, Send Messages, Manage Messages.
  3. Manage Messages is required — the bot deletes the webhook post immediately after reading it so the import key never sits in the channel. If the bot can't delete, it stops the import and notifies the user.

Create the webhook

  1. Open the import channel → Edit Channel → Integrations → Webhooks → New Webhook.
  2. Name it (e.g. "SC Hangar Importer") and copy the URL.
  3. Share the URL with org members so they can paste it into the extension's 🤖 Bot Webhook URL setting.
⚠️ Treat the webhook URL as semi-sensitive. Anyone with it can post to the channel, though they still need a valid import key to trigger an actual import.

Import keys

Each member gets a personal import key automatically when they first open their profile in the Fleet Dashboard. They can view and regenerate it via Personal Profile → 📤 Import. Keys are stored in the bot's database — they are never transmitted except as part of the webhook post, which the bot immediately deletes.

FAQ

Does the extension send my data anywhere automatically?

No. The extension only acts when you click a button. Nothing is sent in the background. Data leaves your device only when you click Send to Bot (to the webhook) or Download JSON (to your local Downloads folder). Opening in Analyzer pushes data directly to the browser tab — nothing goes to a server.

What exactly does the bot receive?

When you click Send to Bot, the extension sends: ships owned, paints owned, blueprints found, session stats (missions, kills, deaths, aUEC earned), and hardware info (CPU, GPU, RAM, OS) from your log files. The webhook post is immediately deleted by the bot — no one in the channel can download your JSON file.

Can someone spoof an import as me?

Not if your import key is secret. The bot validates your personal key on every import. Without it, the upload is rejected. If you think your key was compromised, generate a new one from your Discord profile — the old key is immediately invalidated.

My Send to Bot button is greyed out

All four conditions must be met: hangar data fetched, Discord connected, webhook URL saved, and import key saved. Hover the button to see a tooltip explaining which step is missing.

I'm not receiving a DM after importing

The bot sends import results via Direct Message. Check that your Discord privacy settings allow DMs from server members (User Settings → Privacy & Safety → Allow direct messages from server members). If DMs are off, the bot posts a brief mention in the import channel instead.

The extension worked before but now shows errors

RSI occasionally changes their page structure, which can affect hangar fetching. Check for extension updates from your admin. If Discord Connect fails, try disconnecting and reconnecting — your token may have expired.
